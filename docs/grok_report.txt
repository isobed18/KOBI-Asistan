2. SATICI (İşletme Sahibi / Yönetici) İçin Sunacağımız Değer
Satıcı şu an günde 2-3 saat manuel iş yapıyor. Biz bunu 15-20 dakikaya düşürmeliyiz.
React Dashboard — Ana Sayfalar
A. Overview / Sabah Paneli (İlk açılan)

Günlük özet kartları (sipariş adedi, ciro, bekleyen paket, kritik stok)
LLM Daily Briefing (sabah 08:00 APScheduler ile üretilir)
Riskli durumlar (geciken kargo, düşük stok, iptal talepleri)

B. Sipariş Yönetimi (Task 2)

Tüm siparişler tablosu + filtre
Detay modalı
Human-in-the-Loop:
İptal taleplerini onayla/reddet
Durum manuel güncelle
Müşteriye not / özel mesaj gönder


C. Kargo Yönetimi (Task 3)

Kargo dashboard’u (renk kodlu: Yeşil=normal, Sarı=gecikme riski, Kırmızı=gecikmiş)
Toplu işlem (seçili kargolara bildirim gönder)
Gecikme tespiti + otomatik rapor

D. Stok & Envanter (Task 4)

Ürün listesi + anlık stok + uyarılar
Düşük stok → LLM Yeniden Sipariş Önerisi (geçmiş satışlara göre miktar + tedarikçi taslağı)
Stok hareket geçmişi

E. Analitik & İçgörü (Task 6)

Grafikler (satış trendi, ürün performansı)
LLM-powered Insights (aylık/haftalık)
Rapor export (PDF + Excel)

F. Ticket / Human Review Paneli

Müşteriden gelen iptal, iade, şikayet talepleri
Agent’in çözemediği konular


3. Otomasyon ve Human-in-the-Loop Akışları
Otomatik Yapılacaklar (LLM’siz veya minimum LLM):

Sabah 08:00 → Günlük özet + kritik stok bildirimi (Telegram + Dashboard)
4 saatte bir → Kargo gecikme taraması
Stok eşiğinin altına düşünce → Bildirim + öneri
Yeni sipariş geldiğinde → Müşteriye otomatik onay mesajı (template)
Kargo durumu değiştiğinde → Müşteriye otomatik güncelleme (template)

Human-in-the-Loop Gerekenler (LLM destekli):

Müşteri iptal talebi → Agent ticket oluşturur + LLM özet yazar → Satıcı onaylar
İade talebi
Özel indirim / promosyon talebi
Hasarlı ürün şikayeti
Tedarikçi siparişi onayı (LLM miktar önerir, satıcı onaylar)
Büyük tutarlı siparişlerde manuel onay


4. Multi-Tenant Stratejisi (KOBİ’ler için)
Şimdi yapmayın → Hackathon sonrası düşünün.
Hackathon için:

Tek işletme (single tenant) yeterli.
Kodda tenant_id kolonu ekleyin (geleceğe hazır olsun).
Dashboard ve agent’larda tenant_id filtrelemesi yapın.

Gerçek multi-tenant:

YAML config (her KOBİ kendi prompt kişiselleştirmesi, logo, kuralları)
Ayrı DB schema veya tenant_id bazlı row-level security
